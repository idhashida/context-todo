# Техническое задание

## Цели разработки

Основной целью разработки веб-приложения для управления личными и рабочими задачами является создание удобного и интуитивно понятного инструмента, который повысит продуктивность пользователей за счет организации и упрощения их повседневной деятельности через введение формальных сценариев. Приложение будет сочетать ключевые элементы методологий **GTD (Getting Things Done)** и **BDD (Behaviour-Driven Development)**, предлагая уникальный подход к управлению задачами - объединенную методологию **BDTM (Behaviour-Driven Task Management)**.


## Функциональные требования

### Ключевые сущности системы

#### ⬤ Пользователь

**Пользователь** - конкретная учётная запись со своими реквизитами входа в систему.

#### ⬤ Список

**Список** - сущность системы, которая категоризирует и характеризует задачи. Группа задач, которую можно создать ("Работа", "Личное" и т.д.).

#### ⬤ Подсписок

**Подсписок** - фиксированные группы задач внутри списка (по GTD). Например - "Когда-нибудь", "Проекты", "Следующие действия".
**!!!** Возможно сделаю вид Канбан с переносом задач, если это не сложно. Если же сложно, то тогда перемещение будет реализовано через контекстное меню задачи. **!!!**

#### ⬤ Задача

**Задача** - основная сущность, представляющая собой действие, которое необходимо выполнить.

#### ⬤ Календарь

**Календарь** - сущность системы, которая отображает задачи по дням в виде одной недели.

#### ⬤ Блок "Входящие"

**Список "Входящие"** - специальное хранилище для новых задач без категории (без привязки к конкретному списку).

#### ⬤ Блок "Корзина"

**Блок "Корзина"** - хранилище всех задач, которые были помечены флагом на удаление.

#### ⬤ Статус

**Статус** - определенное состояние задачи на текущий момент.

### Какие действия могут совершаться в системе

* Аутентификация
* Просмотр списка задач
* Перенос задач между списками и подсписками
* Редактирование, создание, удаление и изменение задачи
    - Создание задачи.
    - Изменение параметров задачи.
    - Удаление задачи из списка (перенос задачи в блок "Корзина").
* Просмотр всех задач в виде кадлендаря на 1 неделю

### Функциональные блоки

#### ⬤ Аутентификация / Регистрация (логика входа в личный кабинет / регистрации)

К данному функциональному блоку относятся все процедуры и модули, отвечающие за вход пользователя в личный кабинет системы и регистрацию.

#### ⬤ Авторизация

Модули системы, отвечающие за проверку наличия зарегистрированного пользователя.

#### ⬤ Списки

Модули системы, отвечающие за создание, редактирование и удаление списков задач. Также данный блок отвечает за функционал распределения, визуализации задач на календаре.

#### ⬤ Задачи

Модули системы, отвечающие за редактирование, создание и удаление задач.

#### ⬤ Настройки

Модули системы, отвечающие за базовую настройку приложения - тема приложения, отображаемые элементы (изменяемый дизайн кнопок и т.п.).

### Типы пользователей

#### ⬤ Гость (анонимный пользователь)

Гость не имеет права входа в личный кабинет пользователя. Не может использовать представленный в защищенном разделе сайта функционал. Не может обращатья к внутренним интерфейсам.

#### ⬤ Пользователь

* Может входить в личный кабинет при помощи своей учетной записи
* Может использовать инструменты для управления (создания/ редактирования/ удаления) отдельными сущностями системы

### Доступные интерфейсы системы

#### ⬤ Фронт часть - короткая презентация

Общая информационная страница с логотипом и кратким описанием системы.

Элементы и блоки:
* Хэдер (кликабельный логотип, 2 кнопки - вход и регистрация)
* Тело страницы - краткое описание сервиса
* Футер (копирайт, гитхаб + что-нибудь ещё)

#### ⬤ Страница входа

Страница ввода логина и пароля для входа в защищенную часть сайта (личный кабинет).

Элементы и блоки:
* Хэдер (кликабельный логотип, 1 кнопка - регистрация)
* Форма ввода логина и пароля
* Футер (копирайт, гитхаб + что-нибудь ещё)

#### ⬤ Страница регистрации

Страница ввода почты, логина и пароля для создания личной учетной записи.

Элементы и блоки:
* Хэдер (кликабельный логотип, 1 кнопка - вход)
* Форма ввода почты, логина и пароля
* Футер (копирайт, гитхаб + что-нибудь ещё)

#### ⬤ Сайдбар (сквозная навигация)

Боковое меню (сайдбар) для сквозной навигации по приложению.

Элементы и блоки:
* Аватар пользователя, при нажатии появляется список доступных функций (настройки, выход)
* Кнопка "Добавить задачу" - можно иконку плюсика и т.п. Данная кнопка добавляет новую задачу, которая уходит в папку "Входящие".
* Иконка списка задач
* Иконка календаря

#### ⬤ Списки задач

Страница со всеми задачами пользователя, включает в себя разделение задач по спискам.

Элементы и блоки:
* Блок "Входящие" - все новые задачи попадают сюда
* Блок "Списки" - влючает в себя различные пользовательские списки с возможностью добавления и редактирования
* Блок "Корзина" - удаленные задачи (в базе они никогда не удаляются, только помечаются флагом)

Данный интерфейс предоставляет следующие возможности:
1. Добавление новой задачи
    1. Заголовок (vision level)
    2. Описание (feature level)
    3. Контекст (task level)
    4. Сценарий (task level)
    5. Критерий успеха (task level)
    6. Статус
    7. Дедлайн
2. Удаление конкретной задачи - перемещение задачи в папку "Корзина"
3. Редактирование информации конкретной задачи через карточку задачи
    1. Заголовок (vision level)
    2. Описание (feature level)
    3. Контекст (task level)
    4. Сценарий (task level)
    5. Критерий успеха (task level)
    6. Статус
    7. Дедлайн
4. Просмотр карточки отдельной задачи (боковое окно, которое появляется в правой пустой части приложения)

#### ⬤ Карточка задачи

Отедльное "встроенное" окно, которое появляется с правой стороны экрана при нажатии на конкретную задачу.
Отображает подробную информацию о задаче с возможностью изменения всех доступных полей.

#### ⬤ Календарь

Отображение, визуализация всех задач пользователя на недельном календаре.

Элементы и блоки:
* календарь...
