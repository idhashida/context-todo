# Техническое задание

## Цели разработки

Основной целью разработки веб-приложения для управления личными и рабочими задачами является создание удобного и интуитивно понятного инструмента, который повысит продуктивность пользователей за счет организации и упрощения их повседневной деятельности. Приложение будет сочетать ключевые элементы методологий **GTD (Getting Things Done)** и **TDD (Test-Driven Development)**, предлагая уникальный подход к управлению задачами.

**Основные цели**:

1) Повышение продуктивности пользователей
    - Обеспечить удобный и структурированный способ управления личными и рабочими задачами.
    - Создать возможность быстрого добавления задач и их группировки по категориям, приоритетам и контексту.

2) Применение принципов GTD (Getting Things Done)
    - Реализовать входящий список ("Inbox") для первичной обработки задач.
    - Поддерживать разбиение задач на подзадачи для более точного планирования.
    - Предоставить инструменты для анализа выполненных задач и планирования на будущее (например, еженедельный обзор).

3) Интеграция принципов TDD (Test-Driven Development)
    - Внедрить функциональность проверки задач на соответствие заданным критериям (тестам), чтобы пользователь мог быть уверен в правильности их выполнения.
    - Реализовать возможность автоматизированной проверки задач через интеграции с другими сервисами (например, календарями, файловыми хранилищами).
    - Минималистичный и удобный интерфейс

4) Создать простой и интуитивно понятный дизайн, ориентированный на задачи пользователя.
    - Обеспечить возможность использования приложения на всех устройствах (адаптивный интерфейс).

5) Обеспечение мотивации пользователей
    - Предоставить статистику выполнения задач, включая прогресс, процент успешных завершений, и анализ причин невыполнения задач.
    - Реализовать систему напоминаний для своевременного выполнения задач.

6) Уникальность и конкурентоспособность
    - Объединить в приложении две современные методики (GTD и TDD) для создания уникального продукта, который выделится на фоне традиционных приложений для управления задачами.
    - Подчеркнуть ориентированность на улучшение качества выполнения задач, а не только их учет.

7) Гибкость и простота использования
    - Обеспечить пользователю возможность настроить приложение под свои нужды: от тем интерфейса до уведомлений и напоминаний.
    - Сократить время, необходимое для взаимодействия с приложением, сохраняя его эффективность.



## Функциональные требования

### Ключевые сущности системы

1) **Пользователи (Users)**

_Описание_: Представляют учетные записи пользователей, управляющих своими задачами.

_Основные атрибуты_:
  - user_id (уникальный идентификатор пользователя).
  - email (адрес электронной почты для входа).
  - password_hash (хэш пароля для безопасности).
  - name (имя пользователя).
  - created_at (дата регистрации).

_Функции_:
  - Регистрация и аутентификация.
  - Персонализация задач и настроек.

2) **Задачи (Tasks)**

_Описание_: Главная сущность системы, представляющая задачи пользователя, связанные с личными или рабочими делами.

_Основные атрибуты_:
  - task_id (уникальный идентификатор задачи).
  - user_id (ссылка на пользователя, создавшего задачу).
  - title (название задачи).
  - description (описание задачи).
  - status (статус выполнения: pending, in-progress, completed, failed).
  - context (контекст задачи, например: "Звонки", "Чтение").
  - priority (приоритет задачи: low, medium, high).
  - due_date (срок выполнения).
  - created_at (дата создания задачи).
  - completed_at (дата завершения задачи).

_Функции_:
  - Разделение задач по категориям и контексту.
  - Учет сроков и приоритетов.
  - Ведение истории выполнения задач.

3) **Категории (Categories)**

_Описание_: Объединяет задачи по смысловым группам, таким как личные или рабочие.

_Основные атрибуты_:
  - category_id (уникальный идентификатор категории).
  - user_id (ссылка на пользователя).
  - name (название категории, например: "Личные", "Рабочие").
  - created_at (дата создания категории).

_Функции_:
  - Управление задачами через фильтры по категориям.
  - Организация задач для лучшей структуры.

4) **Подзадачи (Subtasks)**

_Описание_: Представляет этапы выполнения основной задачи, поддерживая методику GTD.

_Основные атрибуты_:
  - subtask_id (уникальный идентификатор подзадачи).
  - task_id (ссылка на основную задачу).
  - title (название подзадачи).
  - status (статус выполнения: pending, in-progress, completed).

_Функции_:
  - Детализация шагов выполнения задачи.
  - Отслеживание прогресса через подзадачи.

5) **Тесты (Tests)**

_Описание_: Представляет собой критерии, которые используются для проверки выполнения задач (методика TDD).

_Основные атрибуты_:
  - test_id (уникальный идентификатор теста).
  - task_id (ссылка на задачу).
  - title (описание теста, например: "Отправить отчет на email").
  - type (тип теста: manual, automatic).
  - status (результат теста: passed, failed, pending).
  - created_at (дата создания теста).

_Функции_:
  - Привязка критериев завершенности к задачам.
  - Автоматическая проверка выполнения (например, интеграция с API).

6) **Напоминания (Reminders)**

_Описание_: Помогает пользователям не пропустить важные задачи или действия.

_Основные атрибуты_:
  - reminder_id (уникальный идентификатор напоминания).
  - task_id (ссылка на задачу).
  - reminder_time (время напоминания).
  - status (статус напоминания: sent, pending).

_Функции_:
  - Автоматическая отправка уведомлений пользователю.
  - Синхронизация с календарем.

7) **Настройки (Settings)**

_Описание_: Позволяет пользователям настроить приложение под свои нужды.

_Основные атрибуты_:
  - settings_id (уникальный идентификатор настройки).
  - user_id (ссылка на пользователя).
  - theme (тема интерфейса: светлая/темная).
  - notification_preferences (настройки уведомлений: включены/выключены).

_Функции_:
  - Управление интерфейсом и уведомлениями.
  - Настройка предпочтений отображения задач.
  - Взаимосвязи сущностей

**Пользователь и задачи**: Один пользователь может иметь много задач, но каждая задача принадлежит только одному пользователю.

**Задачи и подзадачи**: Одна задача может иметь много подзадач.

**Задачи и тесты**: Одна задача может иметь несколько тестов для проверки выполнения.

**Задачи и напоминания**: Одна задача может иметь несколько напоминаний.

**Категории и задачи**: Категория объединяет множество задач, каждая задача относится только к одной категории.

### Какие действия могут совершаться в системе

1) **Управление учетной записью**
  - Регистрация нового пользователя
    - Создание учетной записи с указанием email и пароля.
  - Аутентификация
    - Вход в приложение с использованием учетных данных.
  - Редактирование профиля
    - Изменение имени, email, пароля и других настроек профиля.

2) **Работа с задачами**
  - Добавление задачи
    - Создание новой задачи с указанием названия, описания, контекста, приоритета и срока выполнения.
  - Редактирование задачи
    - Изменение параметров задачи, включая название, описание, категорию, контекст и приоритет.
  - Удаление задачи
    - Удаление задачи из списка (с подтверждением).
  - Перемещение задачи
    - Перемещение задачи из "Inbox" в конкретную категорию или проект.
  - Просмотр списка задач
    - Фильтрация задач по категориям, приоритетам, контексту или статусу выполнения.

3) **Работа с подзадачами**
  - Добавление подзадач
    - Разбиение задачи на более мелкие этапы (например, для планирования шагов).
  - Редактирование подзадач
    - Изменение параметров подзадачи.
  - Отметка выполнения подзадачи
    - Пометка подзадачи как выполненной.

4) **Применение методики GTD (Getting Things Done)**
  - Управление входящим списком ("Inbox")
    - Добавление задач без конкретной категории.
    - Обработка входящих задач и распределение их по категориям и проектам.
  - Контекст задач
    - Привязка задачи к определенному контексту (например, "Звонки", "Чтение").
  - Еженедельный обзор
    - Просмотр выполненных задач за прошедшую неделю.
    - Перенос невыполненных задач в новые временные рамки.

5) **Применение методики TDD (Test-Driven Development)**
  - Добавление тестов к задачам
    - Создание критериев завершенности задачи (например, "Файл отправлен", "Встреча проведена").
  - Редактирование тестов
    - Изменение условий или методов проверки задачи.
  - Автоматическая проверка задач
    - Запуск автоматических тестов (например, через интеграции с календарем или API).
  - Просмотр результатов тестирования
    - Анализ выполненных и проваленных тестов для каждой задачи.

6) **Работа с категориями**
  - Создание категорий
    - Добавление категорий для группировки задач (например, "Рабочие", "Личные").
  - Редактирование категорий
    - Изменение названия или порядка отображения категорий.
  - Удаление категорий
    - Удаление пустой категории (с подтверждением).

7) **Управление напоминаниями**
  - Добавление напоминаний к задачам
    - Установка времени напоминания для задачи.
  - Редактирование напоминаний
    - Изменение времени или удаление напоминаний.
  - Получение уведомлений
    - Уведомления о предстоящих задачах через email или пуш-уведомления.

8) **Аналитика и статистика**
  - Просмотр выполненных задач
    - Список задач, завершенных за определенный период времени.
  - Анализ выполнения тестов
    - Отображение процента успешно пройденных тестов.
  - Обзор продуктивности
    - Отчет о выполненных задачах, просроченных задачах и задачах в процессе выполнения.

9) **Настройки системы**
  - Выбор темы интерфейса
    - Переключение между светлой и темной темами.
  - Настройка уведомлений
    - Управление частотой и форматом уведомлений (например, email или пуш-уведомления).
