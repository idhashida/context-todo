# Техническое задание

## Цели разработки

Основной целью разработки веб-приложения для управления личными и рабочими задачами является создание удобного и интуитивно понятного инструмента, который повысит продуктивность пользователей за счет организации и упрощения их повседневной деятельности через введение формальных сценариев. Приложение будет сочетать ключевые элементы методологий **GTD (Getting Things Done)** и **BDD (Behaviour-Driven Development)**, предлагая уникальный подход к управлению задачами - методологию **BDTM (Behaviour-Driven Task Management)**.



## Функциональные требования

### Ключевые сущности системы

#### ⬤ Пользователь
**Пользователь** - конкретная учётная запись со своими реквизитами входа в систему.
#### ⬤ Задача
**Задача** - основная сущность, представляющая собой действие, которое нужно выполнить.
#### ⬤ Сценарий
**Сценарий** - описание задачи в формате Given-When-Then.
#### ⬤ Контекст
**Контекст** - условия выполнения задач (время, ресурсы и т.д.).
#### ⬤ Критерий успешности
**Критерий успешности** - проверяемое условие завершения задачи (может быть несколько критериев успешности).
#### ⬤ Статус
**Статус** - определенное состояние задачи на текущий момент.

### Какие действия могут совершаться в системе

* Аутентификация
* Управление учетной записью
  - Редактирование профиля пользователя
* Просмотр списка задач
* Редактирование, создание, удаление и изменение задачи
    - Создание задачи.
    - Изменение параметров задачи.
    - Удаление задачи из списка (с подтверждением).
* Редактирование, создание, удаление и изменение статусов задач (исключая неизменяемые статусы)

### Функциональные блоки

#### ⬤ Аутентификация / Регистрация (логика входа в личный кабинет / регистрации)

К данному функциональному блоку относятся все процедуры и модули, отвечающие за вход пользователя в личный кабинет системы и регистрацию.

#### ⬤ Авторизация

Модули системы, отвечающие за проверку наличия зарегистрированного пользователя.

#### ⬤ Профиль пользователя

Модули системы, отвечающие за изменение информации о текущем пользователе (email, пароль, логин).

#### ⬤ Задачи

Модули системы, отвечающие за редактирование, создание и удаление задач.

#### ⬤ Статус

Модули системы, отвечающие за редактирование, создание и удаление статусов задач.

### Типы пользователей

#### ⬤ Гость (анонимный пользователь)

Гость не имеет права входа в личный кабинет пользователя. Не может использовать представленный в защищенном разделе сайта функционал. Не может обращатья к внутренним интерфейсам.

#### ⬤ Пользователь

* Может входить в личный кабинет при помощи своей учетной записи
* Может использовать инструменты для управления (создания/ редактирования/ удаления) отдельными сущностями системы

### Доступные интерфейсы системы

#### ⬤ Фронт часть - короткая презентация

Общая информационная страница с логотипом и кратким описанием системы.

Элементы и блоки:
* Хэдер (кликабельный логотип, 2 кнопки - вход и регистрация)
* Тело страницы - краткое описание сервиса
* Футер (копирайт, гитхаб + что-нибудь ещё)

#### ⬤ Страница входа

Страница ввода логина и пароля для входа в защищенную часть сайта (личный кабинет).

Элементы и блоки:
* Хэдер (кликабельный логотип, 2 кнопки - вход и регистрация)
* Форма ввода логина и пароля
* Футер (копирайт, гитхаб + что-нибудь ещё)

#### ⬤ Профиль пользователя (защищенная часть сайта)

Страница с основной информацией о текущем пользователе.

Элементы и блоки:
* Логин
* Email
* Пароль

#### ⬤ Список задач

Страница со всеми "активными" задачами пользователя.

Элементы и блоки:
*

1. Добавление новой задачи
    1. Заголовок
    2. Описание
    3. Контекст
    4. Сценарий
    5. Критерий успеха
    6. Статус

3. Редактирование информации конкретной задачи

    6. Статус

4. Просмотр карточки отдельной задачи (модальное окно?)

Отдельное окно с информацией о задаче...

#### ⬤ Список доступных статусов

Страница со всеми статусами, включая неизменяемые системные.

Элементы и блоки:
* Наименование статуса
* Кнопка для изменения наименования

Данный интерфейс предоставляет следующие возможности:
1. Добавление пользовательского статуса
    1. Наименование
2. Удаление пользовательского статуса
3. Редактирование пользовательского статуса
    1. Наименование
4. Просмотр списка всех возможных статусов
